## Project Overview

**Kreitz Research Group** website, hosted on GitHub Pages.
**Stack:** Jekyll 4.x · custom CSS (Niemeyer-inspired sidebar layout) · `remote_theme: pages-themes/minimal`
**Content model:** Markdown pages + `_data/` YAML files
**Automation:** Python scripts for publication updates

Deployment is automatic: push to `main` → GitHub Actions builds with Jekyll → GitHub Pages.

## Design System

### Layout
Sidebar + content, mirroring niemeyer-research-group.github.io:
- **Sidebar** (15em, sticky): PI photo, name, title, nav links, contact icons
- **Content** (max 40em): page content with 10em top margin on desktop
- **Responsive**: stacks to single column below 800px; nav becomes horizontal pills

### Typography
- Headers: **Poppins** 700 (Google Fonts), scale from 20pt (h1) down by 3pt per level
- Body: **Nunito Sans** 300/600 (Google Fonts), 14pt, line-height 1.5em

### Colors (CSS variables in `assets/css/main.css`)
- `--bg` / `--text`: white/black (dark mode: inverted)
- `--link`: `#0000b8` (dark mode: `#a2a2ff`)
- `--muted`: `#555555` — used for author names, metadata
- `--border`: `#e8e8e8` — card borders, hr, h2 underlines

Dark mode is supported via `@media (prefers-color-scheme: dark)`.

## Content Data Schemas

### `_data/people.yml`
```yaml
- id: firstname-lastname
  name: "First Last"
  role: "Ph.D. Student"
  status: current          # pi | current | alumni
  photo: "first-last.jpg"  # in assets/images/people/
  education: "B.S. ChE (University, year)"
  info: ""                 # awards, short bio
  year_joined: 2024
  current_position: ""     # for alumni: where they are now
  email: ""
  github: ""
  google_scholar: ""
  linkedin: ""
  cv_url: ""
```

### `_data/publications.yml`
Auto-generated by scripts. Manual override fields (never overwritten on re-run):
`featured`, `image`, `corresponding_author`, `preprint`, `abstract`,
`catalysis_surface_science`, `data_science`, `computational_chemistry`

### `_data/repos.yml`
```yaml
- name: "ToolName"
  description: "One-sentence description."
  url: "https://github.com/kreitz-group/toolname"
  docs_url: ""
```

## Publication Scripts

Two-phase workflow in `scripts/`. See `scripts/README.md` for full details.

```bash
cd scripts
source .venv/bin/activate          # or: python -m venv .venv && pip install -r requirements.txt
python fetch_publications.py --orcid YOUR_ORCID --author-name "Kreitz"
export ANTHROPIC_API_KEY=sk-ant-...
python clean_publications.py
# Review diff, then commit _data/publications.yml
```

## Common Maintenance Tasks

**Add a new member:** add entry to `_data/people.yml` (status: current) + photo in `assets/images/people/`
**Member graduates/leaves:** change `status: current` → `status: alumni`, set `current_position`
**Add a software tool:** add entry to `_data/repos.yml`
**Update publications:** run the two-phase script workflow above
**Update PI info:** edit the `pi:` block in `_config.yml`
